28 ## Usage
29 
30 **Initialization**
31 
32 Create a Serial Terminal object. This can be initialized with optional newline and delimiter characters.
33 
34 Default newline character: ```'\n'```
35 Default delimiter character: ```Space```
36 
37 ```c++
38 #include <ErriezSerialTerminal.h>
39 
40 // Newline character '\r' or '\n'
41 char newlineChar = '\n';
42 // Separator character between commands and arguments
43 char delimiterChar = ' ';
44 
45 // Create serial terminal object
46 SerialTerminal term(newlineChar, delimiterChar);
47 
48 
49 void setup()
50 {
51  // Initialize serial port
52  Serial.begin(115200);
53 
54  // Initialize the built-in LED
55  pinMode(LED_BUILTIN, OUTPUT);
56  digitalWrite(LED_BUILTIN, LOW);
57 }
58 ```
59 **Register new commands**
60 
61 Commands must be registered at startup with a corresponding ```callback handler``` . This registers the command only, excluding arguments.
62 
63 The callback handler will be called when the command has been received including the newline character.
64 
65 An example of registering multiple commands:
66 
67 ```c++
68 void setup()
69 {
70  ...
71 
72  // Add command callback handlers
73  term.addCommand("?", cmdHelp);
74  term.addCommand("help", cmdHelp);
75  term.addCommand("on", cmdLedOn);
76  term.addCommand("off", cmdLedOff);
77 }
78 
79 void cmdHelp()
80 {
81  // Print usage
82  Serial.println(F("Serial terminal usage:"));
83  Serial.println(F(" help or ? Print this usage"));
84  Serial.println(F(" on Turn LED on"));
85  Serial.println(F(" off Turn LED off"));
86 }
87 
88 void cmdLedOn()
89 {
90  // Turn LED on
91  Serial.println(F("LED on"));
92  digitalWrite(LED_BUILTIN, HIGH);
93 }
94 
95 void cmdLedOff()
96 {
97  // Turn LED off
98  Serial.println(F("LED off"));
99  digitalWrite(LED_BUILTIN, LOW);
100 }
101 ```
102 
103 **Set default handler**
104 
105 Optional: The default handler will be called when the command is not recognized.
106 
107 ```c++
108 void setup()
109 {
110  ...
111 
112  // Set default handler for unknown commands
113  term.setDefaultHandler(unknownCommand);
114 }
115 
116 void unknownCommand(const char *command)
117 {
118  // Print unknown command
119  Serial.print(F("Unknown command: "));
120  Serial.println(command);
121 }
122 ```
123 
124 **Read from serial port**
125 
126 Read from the serial port in the main loop:
127 
128 ```c++
129 void loop()
130 {
131  // Read from serial port and handle command callbacks
132  term.readSerial();
133 }
134 ```
135 
136 **Get next argument**
137 
138 Get pointer to next argument in serial receive buffer:
139 
140 ```c++
141 char *arg;
142 
143 // Get next argument
144 arg = term.getNext();
145 if (arg != NULL) {
146  Serial.print(F("Argument: "));
147  Serial.println(arg);
148 } else {
149  Serial.println(F("No argument"));
150 }
151 ```
152 
153 **Get remaining characters**
154 
155 Get pointer to remaining characters in serial receive buffer:
156 
157 ```c++
158 char *arg;
159 
160 // Get remaining characters
161 arg = term.getRemaining();
162 if (arg != NULL) {
163  Serial.print(F("Remaining: "));
164  Serial.println(arg);
165 }
166 ```
167 
168 **Clear buffer**
169 
170 Optional: The serial receive buffer can be cleared with the following call:
171 
172 ```c++
173 term.clearBuffer();
174 ```

177 ## Library configuration
178 
179 ```SerialTerminal.h``` contains the following configuration macro's:
180 
181 * ```ST_RX_BUFFER_SIZE``` : The default serial receive buffer size is 32 Bytes. This includes the command and arguments, excluding the ```'\0'``` character.
182 * ```ST_NUM_COMMAND_CHARS```: The default number of command characters is 8 Bytes, excluding the ```'\0'``` character.
